A busca por conhecimento em algoritmo e lógica de programação tem crescido significativamente. Esses são temas essenciais em áreas em expansão, oferecendo vastas oportunidades para quem os domina. Computadores não interpretam comandos subjetivos, o que torna a lógica de programação e os algoritmos fundamentais para uma comunicação eficaz com as máquinas. Enquanto a lógica de programação estabelece a estrutura de conceitos e regras para a execução, o algoritmo é a sequência de comandos claros e lógicos que guiam a resolução de problemas ou o alcance de objetivos. Compreender esses conceitos é crucial para desenvolver o raciocínio lógico, indispensável no ambiente tecnológico. Algoritmos são comumente implementados em diversas linguagens de programação, como JavaScript, Java e Python. Apesar das variações de sintaxe entre elas, a lógica permanece mesma, por isso a importância de aprender-la.

O que é a logica de programação ?

Para uma compreensão prática da lógica de programação, considere o objetivo de "ligar o chuveiro" como o algoritmo. Sua função é fornecer as instruções detalhadas para que essa ação seja executada. Assumindo que o sistema já possui habilidades básicas de locomoção, os passos seriam:

  • Selecionar a Temperatura Desejada
  • Abrir o Registro de Água
  • Conferir se está quente
  • Está Ligado!

Entrada | Saida | Processo -

Além da sequência de ações, a programação envolve os conceitos de Entrada e Saída de dados. A entrada compreende todas as informações necessárias para que o computador execute a tarefa proposta. A saída é o resultado gerado por esse processo. Vamos atualizar o exemplo do chuveiro para mostrar:

Entrada -

  • Verificar a Conexão Elétrica (Disjuntor):
  • Verificar Pressão da Agua
  • Verificar Funcionamento do Chuveiro

Processo -

  • Selecionar a Temperatura Desejada
  • Abrir o Registro de Água

Saída -

  • Chuveiro Ligado e Saindo Água quente

SE | SENÃO | ENTÃO -

As estruturas SE, SENÃO e ENTÃO são elementos cruciais na programação, pois permitem a implementação de condições para a execução de ações. Observe o exemplo a seguir, mantendo a lógica do chuveiro:

Exemplo -

  • (Verificar se o Disjuntor estar funcionado)
  • SE (Disjuntor = OK) ENTÃO Prossiga.
  • SENÃO Procure um eletriscista para arrumar a conexão

  • (Verificar se Pressão da Agua esta funcionando)
    • SE (Pressão = OK) ENTÃO Prossiga.
  • SENÃO Arrume a Pressao

  • (Verificar Funcionamento do Chuveiro)
    • SE (Funcionamento = OK) ENTÃO Prossiga.
  • SENÃO Arrume o chuveiro

Variáveis -

Termos como "DISJUNTOR", "PRESSÃO" e "FUNCIONAMENTO", utilizados nos exemplos, são denominados variáveis. Como o próprio nome indica, seus valores podem variar, ou seja, podem apresentar diferentes estados. É necessário atribuir valores a essas variáveis. O "OK" utilizado representa um valor atribuído. Veja um exemplo:

Minhas Variaveis:

  • -Disjuntor = Caracter
  • -Pressão = Caracter
  • -Funcionamento = Caracter

Valores

  • (Disjuntor = OK)
  • (Pressão = OK)
  • (Funcionamento = OK)

Neste cenário, todas as variáveis receberam o valor "OK", indicando que o processo deve ocorrer sem interrupções. Tambem elas se encontram como CARACTER, ou seja os valores a serem atriubuidos serão APENAS letras, se fossem numeros poderia ser atribuido como, Real ou Inteiro, seguindo o padrao dos conjuntos numericos. Caso um valor seja diferente de "OK", uma mensagem de erro será exibida, como por exemplo: "Procure um eletricista para arrumar a conexão".
